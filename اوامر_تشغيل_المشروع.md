# أوامر تشغيل مشروع Smart Study Assistant

---

## 1. المرة الأولى بعد فتح المشروع

افتح **VS Code Terminal** ثم نفّذ الأوامر التالية بالترتيب:

```bash
# الانتقال إلى مجلد المشروع
cd C:\Users\m\smart_study_assistant

# تحميل الحزم والمكتبات
flutter pub get

# فحص الأخطاء البرمجية
flutter analyze

# تشغيل التطبيق على Chrome
flutter run -d chrome
```

---

## 2. التشغيل العادي في المرات القادمة

```bash
cd C:\Users\m\smart_study_assistant
flutter run -d chrome
```

---

## 3. أوامر مفيدة للفحص

```bash
# فحص البيئة والأدوات المثبتة
flutter doctor

# عرض الأجهزة المتاحة للتشغيل
flutter devices

# عرض إصدار Flutter المثبت
flutter --version
```

---

## 4. ملاحظات مهمة

- **إذا لم يتعرف الترمينال على أمر flutter:**
  أغلق VS Code بالكامل ثم أعد فتحه، لأن مسار Flutter يُحمَّل عند بدء التشغيل فقط.

- **Android Studio غير مطلوب:**
  التطبيق يعمل على Chrome مباشرةً بدون الحاجة إلى تثبيت Android Studio.

- **إذا ظهرت أخطاء في flutter analyze:**
  يجب إصلاح جميع الأخطاء قبل تشغيل التطبيق.

- **لا تنفّذ أوامر الحذف أو إعادة الضبط:**
  أوامر مثل `flutter clean` أو `git reset` لا تُستخدم إلا باتفاق الفريق كاملاً.

---

## 5. مسار Flutter SDK

```
C:\src\flutter\bin
```

تمت إضافته إلى **User PATH** تلقائياً — لا تحتاج إلى تعديله يدوياً.

---

## 6. مفاتيح مفيدة أثناء تشغيل التطبيق

| المفتاح | الوظيفة |
|---------|---------|
| `r` | Hot Reload — تطبيق التعديلات فوراً بدون إعادة تشغيل |
| `R` | Hot Restart — إعادة تشغيل كاملة |
| `q` | إيقاف التطبيق |
